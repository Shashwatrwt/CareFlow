# CareFlow: Smart Health Response Platform

A JavaFX desktop application for managing patient registration, bed allocation, and ambulance routing across Uttarakhand, India. It features an interactive Leaflet map, a severity-based priority queue, Dijkstra's shortest-path algorithm for routing, and undo functionality.

## Project Overview

CareFlow is a smart health response platform built as a desktop GUI application. It allows hospital staff to:

1. **Register Patients** with severity levels (`CRITICAL`, `HIGH`, `MEDIUM`, `LOW`).
2. **Allocate Beds** from a pool of 10 beds using a priority queue (critical patients first).
3. **Discharge Patients** and free up beds.
4. **Dispatch Ambulances** between Uttarakhand cities using shortest-path routing.
5. **Undo** the last bed allocation action.

The map view uses **Leaflet.js** inside a JavaFX `WebView` to display cities and draw ambulance routes.

---

## Features

| Feature | Description |
|---------|-------------|
| Patient Registration | Add patients with name, age, and severity. Duplicate names are rejected. |
| Priority Queue | Patients are automatically ordered by severity (`CRITICAL` > `HIGH` > `MEDIUM` > `LOW`). |
| Bed Allocation | Assigns the first available bed from 10 pre-initialized beds. |
| Discharge | Frees the bed and updates patient status to `DISCHARGED`. |
| Ambulance Routing | Uses Dijkstra's algorithm to compute shortest driving routes between 10 Uttarakhand towns. |
| Interactive Map | Displays cities and routes on a Leaflet map (CartoDB Voyager tiles). |
| Undo | Reverts the most recent bed allocation, releasing the bed and restoring patient status. |

## Prerequisites

1. **Java Development Kit (JDK) 17 or higher** installed.
2. **Maven 3.8+** installed and added to your system `PATH`.
   - Download: https://maven.apache.org/download.cgi
   - Installation guide: https://maven.apache.org/install.html
3. **(Optional)** A JavaFX-bundled JDK such as:
   - [Azul Zulu FX](https://www.azul.com/downloads/?package=jdk-fx)
   - [BellSoft Liberica Full](https://bell-sw.com/pages/downloads/)

> **Note:** Standard Oracle/OpenJDK distributions do **not** include JavaFX. If you use a standard JDK, you must use Maven or download the JavaFX SDK separately.

## How to Run Manually

### Option A: Using Maven (Recommended)

This is the easiest method because Maven automatically downloads JavaFX and PostgreSQL drivers.

1. **Open a terminal** in the project root folder (`c:/Users/Mohit/Documents/cf` or wherever you cloned it).

2. **Compile and run:**
   ```bash
   mvn clean javafx:run
   ```

   - `clean` deletes old compiled files.
   - `javafx:run` compiles the code, resolves dependencies, and launches `ui.MainApp`.

3. **Wait for dependencies.** On the first run, Maven downloads JavaFX and PostgreSQL jars. This may take 1–3 minutes depending on your internet speed.

4. **The application window should appear** titled *"CareFlow: Smart Health Response Platform"*.

---

### Option B: Using a JavaFX-Bundled JDK

If you have a JDK that includes JavaFX (e.g., Azul Zulu FX or BellSoft Liberica Full):

1. **Ensure your `JAVA_HOME` points to the JavaFX-bundled JDK.**

2. **Open a terminal** in the project root.

3. **Compile with Maven (or javac):**
   ```bash
   mvn clean compile
   ```

4. **Run manually:**
   ```bash
   java -cp "target/classes" ui.MainApp
   ```

   Or on Windows Command Prompt:
   ```cmd
   java -cp "target/classes" ui.MainApp
   ```

---

### Option C: Using JavaFX SDK + Command Line

If you cannot install Maven but have downloaded the [JavaFX SDK](https://gluonhq.com/products/javafx/):

1. **Download the JavaFX SDK** for your OS and extract it (e.g., to `C:\javafx-sdk-21`).

2. **Open a terminal** in the project root.

3. **Compile the source files manually:**
   ```cmd
   javac --module-path "C:\javafx-sdk-21\lib" --add-modules javafx.controls,javafx.web -d target/classes src/main/java/**/*.java
   ```

4. **Copy resources:**
   ```cmd
   xcopy /E /I src\main\resources target\classes
   ```

5. **Run the application:**
   ```cmd
   java --module-path "C:\javafx-sdk-21\lib" --add-modules javafx.controls,javafx.web -cp "target/classes" ui.MainApp
   ```

## Database Notes

- **Current Implementation:** The `PatientDAO` and `BedDAO` classes use **in-memory `ArrayList`s** for storage. No external database is required to run the application in its current state.
- **Future Integration:** `DBConnection.java` contains placeholder credentials for a **Supabase PostgreSQL** instance. To connect to a real database:
  1. Update `DBConnection.java` with your actual database URL, username, and password.
  2. Modify `PatientDAO` and `BedDAO` to execute SQL queries using `DBConnection.getConnection()` instead of operating on in-memory lists.

## Troubleshooting

| Problem | Solution |
|---------|----------|
| `'mvn' is not recognized` | Install Maven and add its `bin` folder to your system `PATH`. |
| `NoClassDefFoundError: javafx/application/Application` | You are using a standard JDK without JavaFX. Use **Option A (Maven)** or install a **JavaFX-bundled JDK**. |
| `Error: Could not find or load main class ui.MainApp` | Ensure you are running from the project root and the classpath includes `target/classes`. |
| Map does not load / blank map | The app requires an internet connection to load Leaflet and CartoDB tiles. Check your network connection. |
| `Failed to initialize beds` | This is a non-fatal warning. Beds are initialized in `BedService` constructor; if it fails, restart the app. |

## Application Usage Quick Guide

Once the app is running:

1. **Register Patient:** Click *Register Patient* → enter name, age, severity → click *Register*.
2. **Allocate Bed:** Click *Allocate Bed* → the highest-severity waiting patient gets the next free bed.
3. **Show Patients:** Click *Show Patients* to refresh the patient table.
4. **Discharge:** Select a patient in the table → click *Discharge*.
5. **Dispatch Ambulance:** Click *Dispatch Ambulance* → select source and destination cities → click *Dispatch*. The route appears on the map.
6. **Undo:** Click *Undo* to revert the last bed allocation.
